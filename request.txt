انواع درخواست‌های HTTP (HTTP Methods) و کاربرد آنها

==================================================

۱. GET
- کاربرد: دریافت داده از سرور
- توضیح: برای خواندن اطلاعات استفاده می‌شود. این درخواست فقط داده را می‌خواند و نباید اثری روی داده‌ها بگذارد.
- مثال: بارگذاری یک صفحه وب، دریافت اطلاعات کاربر

۲. POST
- کاربرد: ایجاد یک منبع جدید روی سرور
- توضیح: برای ارسال داده‌ها به سرور جهت ایجاد رکورد جدید استفاده می‌شود.
- مثال: ثبت نام کاربر، ارسال فرم

۳. PUT
- کاربرد: به‌روزرسانی کامل یک منبع موجود
- توضیح: برای به‌روزرسانی یک منبع استفاده می‌شود. کل منبع با داده‌های ارسالی جایگزین می‌شود.
- مثال: به‌روزرسانی کامل پروفایل کاربر

۴. PATCH
- کاربرد: به‌روزرسانی جزئی یک منبع موجود
- توضیح: فقط فیلدهای خاصی که تغییر کرده‌اند ارسال و به‌روز می‌شوند.
- مثال: تغییر فقط رمز عبور کاربر

۵. DELETE
- کاربرد: حذف یک منبع از سرور
- توضیح: برای حذف یک منبع مشخص شده استفاده می‌شود.
- مثال: حذف یک پست از وبلاگ

۶. HEAD
- کاربرد: دریافت متادیتا یک منبع
- توضیح: مانند GET عمل می‌کند اما فقط هدرهای پاسخ را بدون بدنه برمی‌گرداند.
- مثال: بررسی وجود یک منبع، وضعیت کش

۷. OPTIONS
- کاربرد: دریافت متدهای مجاز برای یک منبع
- توضیح: مشخص می‌کند کدام متدهای HTTP برای یک URL مجاز هستند.
- مثال: استفاده در مکانیسم CORS

۸. CONNECT
- کاربرد: ایجاد یک تونل به سرور مقصد
- توضیح: برای برقراری ارتباط دوطرفه (مانند SSL/TLS) استفاده می‌شود.

۹. TRACE
- کاربرد: عیب‌یابی و تشخیص
- توضیح: برای تست و مشاهده تغییراتی که توسط سرورهای میانی ایجاد می‌شود.

==================================================

خلاصه ویژگی‌ها:

GET     - دریافت    - Idempotent - Safe
POST    - ایجاد     - Not Idempotent - Not Safe
PUT     - جایگزینی - Idempotent - Not Safe
PATCH   - به‌روزرسانی - Idempotent* - Not Safe
DELETE  - حذف      - Idempotent - Not Safe
HEAD    - متادیتا  - Idempotent - Safe
OPTIONS - متدهای مجاز - Idempotent - Safe

==================================================

توضیح اصطلاحات:
- Idempotent: اجرای چندباره درخواست نتیجه یکسانی دارد
- Safe: درخواست فقط برای خواندن است و تغییری ایجاد نمی‌کند

این متدها اساس معماری RESTful API را تشکیل می‌دهند.
